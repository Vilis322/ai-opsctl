# AI OpsCtl — ML Analytics Platform

> **Synthetic Data Notice:** All data in this project is 100% synthetic, generated for demonstration and educational purposes. No real company data, personal information, or proprietary datasets are used. All names, domains, financial figures, and market data are fictitious.

AI-powered analytics platform with RAG search, iterative fine-tuning pipeline, and versioned dataset management. Part of the [OpsCtl](https://opsctl.tech) ecosystem.

**Live demo:** [ai.opsctl.tech](https://ai.opsctl.tech) (login: `admin` / `admin`)

---

## Architecture

```
React 18 + Tailwind          NestJS API Gateway          FastAPI ML Engine
     :5173                        :3000                       :8100
       |                            |                           |
       +--- /api/* --------------->  +--- /ml/* --------------> |
       |    (REST + SSE streaming)   |    (HTTP proxy)          |
       |                            |                           |
       |    PostgreSQL :5432        |    ChromaDB :8000         |
       |    (Prisma ORM)            |    (Vector store)         |
       |                            |                           |
       +----------------------------+------ MLX / Ollama ------+
                                            (Inference + LoRA fine-tuning)
```

| Layer | Stack | Responsibility |
|-------|-------|---------------|
| **Frontend** | React 18, Vite, Tailwind CSS, Recharts, Socket.IO, i18n (EN/RU/UA) | Chat interface, dashboards, dataset management, training UI |
| **Backend** | NestJS, Prisma, PostgreSQL, JWT, Socket.IO | API gateway, auth, CRUD, real-time, ML proxy |
| **ML Engine** | FastAPI, MLX, LangChain, ChromaDB, Sentence-Transformers | Inference, RAG pipeline, fine-tuning orchestration |
| **Training** | MLX LoRA/QLoRA, PEFT, Hugging Face Transformers, Llama 3.1 8B | Iterative fine-tuning on Apple Silicon (M4 Max) |
| **Infrastructure** | Docker Compose, nginx, Let's Encrypt | PostgreSQL, ChromaDB, reverse proxy, SSL |

## Features

### Chat with AI Model
- Conversational interface with streaming responses (SSE)
- RAG-powered answers grounded in dataset context
- Model/dataset selection per conversation
- Markdown rendering (tables, code blocks, lists)

### Dashboard & Analytics
- Metric cards: leads, revenue, expenses, ROI, active domains
- Charts: timeline trends, buyer performance, vertical distribution, geo heatmap
- Filterable tables with sorting
- All visualizations driven by selected dataset version

### Dataset Versioning
- Deterministic synthetic data generation (seed-based reproducibility)
- Semver versioning (v1.0.0, v1.1.0, v2.0.0)
- Side-by-side comparison between dataset versions
- 3 verticals: SaaS, E-commerce, Fintech across 9 geos
- 12 months of data with seasonality patterns

### Iterative Model Training
- MLX-native LoRA fine-tuning on Apple Silicon
- Training run management: hyperparameters, metrics, status tracking
- Side-by-side run comparison (loss curves, eval metrics)
- Adapter versioning tied to dataset versions
- Ollama fallback for x86 deployment

## Synthetic Dataset

All data is generated by a deterministic seed generator (`ml/seeds/generator.py`):

| Entity | Count | Details |
|--------|-------|---------|
| Buyers | 15 | 3 teams (SaaS, E-commerce, Fintech), 1 fired at month 8 |
| Domains | 200 | Brandable names, 5 TLDs (.com/.io/.co/.app/.dev), 5% banned |
| Servers | 15 | 4 providers (DigitalOcean, Hetzner, Linode, Vultr) |
| Offers | 40+ | 12 base offers x multiple geos, CPL/CPA payout models |
| Leads | 30,000 | Seasonality, geo-appropriate names (Faker), 85/10/5 CRM status |
| Incomes | ~500 | Aggregated from leads x payout, +/-10% variance |
| Expenses | ~1,000 | 5 categories per buyer per month + server costs |
| Training pairs | 20+ | Auto-generated Q&A JSONL for fine-tuning |

## Quick Start

```bash
# Clone and configure
cp .env.example .env

# Start infrastructure
make infra              # PostgreSQL + ChromaDB (Docker)

# Database setup
make db-migrate         # Prisma migrations (15 tables)
make db-seed            # Generate synthetic dataset v1.0.0 (30K leads)

# Development servers (run in separate terminals)
make dev-backend        # NestJS on :3000
make dev-ml             # FastAPI on :8100
make dev-frontend       # React on :5173
```
