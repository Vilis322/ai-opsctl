# CLAUDE.md — AI OpsCtl

## Overview

AI-powered analytics platform with RAG, iterative fine-tuning, and versioned dataset management. Demo/portfolio project with 100% synthetic data. Part of OpsCtl ecosystem (`ai.opsctl.tech`).

## Quick Reference

| | Stack | Docs |
|---|---|---|
| **Frontend** | React 18, TypeScript, Vite, Tailwind, Recharts, i18n (en/ru/ua) | `frontend/` (Phase 4) |
| **Backend** | NestJS, Prisma 7, PostgreSQL, JWT, Socket.IO | `backend/` |
| **ML Engine** | FastAPI, MLX, LangChain, ChromaDB, Sentence-Transformers | `ml/` |
| **Training** | MLX LoRA/QLoRA, PEFT, Llama 3.1 8B, HF Transformers | `ml/app/training/` |

**Domain:** ai.opsctl.tech
**Branch:** `stage` (working), `main` (protected)
**Database port:** 5433 (Docker maps 5433->5432, local PG on 5432)

## CRITICAL: Synthetic Data

**All data is 100% synthetic.** No real company data, personal information, or proprietary datasets. NDA-safe. Generated by `ml/seeds/generator.py` with deterministic seed.

- Buyer tags: Maxwell, Harper, Quinn, etc. (not real people)
- Domains: brandable generated names (cloudhub.io, datanest.com)
- Financials: realistic patterns but fictitious amounts
- Leads: Faker-generated names per geo locale

## Architecture

```
React (:5173) -> NestJS (:3000) -> FastAPI (:8100) -> MLX/Ollama
                      |                  |
                 PostgreSQL          ChromaDB
                  (Prisma)          (Vectors)
```

NestJS = API gateway (auth, CRUD, ML proxy). FastAPI = ML-only (inference, training, RAG). NestJS proxies SSE streaming from FastAPI to React transparently.

## Database (Prisma, 15 tables)

### Auth
- `users` — login, password_hash, language

### Chat
- `conversations` — user_id, title, model_version, dataset_version
- `messages` — conversation_id, role, content, tokens, latency

### Datasets (versioned)
- `datasets` — name, version (semver), status (DRAFT/ACTIVE/ARCHIVED), generator_params
- `dataset_metadata` — key/value per dataset

### Fake CRM (all prefixed `ds_`, FK to dataset_id)
- `ds_buyers` — tag, name, team, vertical, is_active
- `ds_servers` — ip, provider, location, monthly_cost
- `ds_domains` — domain_name, server_id, buyer_id, status, geo, tld
- `ds_offers` — name, vertical, geo, conversion_rate, payout_amount
- `ds_leads` — domain_id, offer_id, buyer_id, geo, contact info, crm_status
- `ds_expenses` — buyer_id, category, amount, date
- `ds_incomes` — buyer_id, offer_id, geo, type (cpl/cpa/crg), amount, date

### ML
- `training_runs` — dataset_id, model_base, adapter_name, hyperparams, metrics, status
- `training_comparisons` — run_a_id, run_b_id, notes
- `rag_collections` — dataset_id, name, doc_count, embedding_model, status

## Seed Generator

```bash
cd ml && python seeds/generator.py --version v1.0.0 --months 12 --leads 30000 --seed 42
```

Deterministic (seed=42). Generates: 15 buyers, 15 servers, 200 domains, 40+ offers, 30K leads, ~500 incomes, ~1K expenses, 20+ training Q&A pairs.

Exports: PostgreSQL records + `data/datasets/v1.0.0/training.jsonl` + `summary.json`.

## ML Stack

- **MLX/MLX-LM** — Apple Silicon native fine-tuning (LoRA) + inference
- **Ollama** — fallback for x86 VPS (`INFERENCE_BACKEND` auto-detected)
- **LangChain** — RAG orchestration (document loading, splitting, retrieval chains)
- **ChromaDB** — vector store for embeddings
- **Sentence-Transformers** — embedding models (all-MiniLM-L6-v2)
- **PEFT** — LoRA adapter management
- **HF Transformers** — tokenizers, model hub

## Dev Commands

```bash
make infra          # Start PostgreSQL + ChromaDB
make infra-down     # Stop infrastructure
make db-migrate     # Prisma migrate dev
make db-seed        # Generate dataset v1.0.0
make db-reset       # Reset DB
make dev-backend    # NestJS :3000
make dev-ml         # FastAPI :8100
make dev-frontend   # React :5173
make test           # Run all tests
make clean          # Nuke volumes + data
```
